# lew-ui AI 协作指南

## 项目定位

lew-ui 是一个现代化、轻量级的 Vue 3 组件库，专注于提供简洁、优雅且高性能的 UI 组件。

## 设计哲学

### 1. 简洁至上
- **API 设计简单明了**：每个组件的 props 和 events 应该直观易懂
- **代码风格简洁**：避免过度抽象，优先考虑可读性
- **文档简明扼要**：提供清晰的示例和说明

### 2. 性能优先
- **按需加载**：支持 tree-shaking，用户只引入需要的组件
- **最小体积**：每个组件都应该追求最小的打包体积
- **运行时高效**：避免不必要的响应式数据和重复渲染

### 3. 开发者友好
- **TypeScript 优先**：完整的类型支持和类型推断
- **一致性**：统一的组件 API 设计模式
- **可扩展性**：通过 props、slots、CSS 变量提供灵活的定制能力

### 4. 用户体验
- **视觉美观**：现代化的设计风格
- **交互流畅**：合理的过渡动画和加载状态
- **无障碍访问**：考虑可访问性标准

## AI 协作原则

### 编码风格
- 使用 Vue 3 Composition API (`<script setup>`)
- TypeScript 严格模式
- 遵循 ESLint 和 Stylelint 配置
- 使用 SCSS 编写样式

### 响应模式
1. **理解需求**：先理解用户意图，必要时提出问题
2. **查看规范**：参考 `.cursor/rules/` 下的具体规范文件
3. **保持一致**：遵循现有代码风格和组件结构
4. **完整实现**：不要留下 TODO 或半成品代码
5. **自我检查**：确保代码符合 TypeScript 类型检查和 Linter 规则

### 修改策略
- **谨慎修改公共 API**：参考 `public-api.mdc`
- **保持向后兼容**：除非是 breaking change 并明确标注
- **同步更新文档**：代码变更要同步更新相关文档
- **测试覆盖**：确保关键逻辑有测试用例

## 关键约束

- Node.js 版本：^22.0.0
- Vue 版本：^3.5.3
- 构建工具：Vite
- 包管理器：pnpm
- 代码规范：@antfu/eslint-config

## 常见任务

### 新建组件
参考 `rules/component-structure.mdc`，遵循标准的组件目录结构。

### 修改现有组件
1. 检查 `rules/public-api.mdc` 确保不破坏 API
2. 运行 `pnpm lint:fix` 和 `pnpm lint:css:fix`
3. 更新类型定义
4. 运行 `pnpm typecheck` 检查类型

### 样式调整
参考 `rules/style-system.mdc`，使用 CSS 变量系统和主题化方案。

### 性能优化
参考 `rules/vue3-performance.mdc`，关注组件体积和运行时性能。

---

**记住**：你是 lew-ui 的协作者，目标是帮助创建一个高质量、易用、美观的组件库。
